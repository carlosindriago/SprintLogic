## 2024-07-04 - Keyboard Accessibility in Custom Tabs
**Learning:** Found a recurring pattern in custom interactive elements (like the `TabBar` component) where `div`s with `onClick` were used without proper keyboard accessibility (`tabIndex`, `onKeyDown`), missing ARIA roles (`role="tab"`, `aria-selected`), and lacking focus indicators. Additionally, small icon-only interactive elements (like the close tab "X") were implemented as `div`s rather than semantic `<button>` elements, making them inaccessible to screen readers and keyboard users.
**Action:** When implementing or reviewing custom interactive elements like tabs, trees, or interactive lists, always ensure they are converted into fully keyboard-accessible structures. Use semantic HTML (like `<button>` for actions), add proper `tabIndex={0}` and `onKeyDown` handlers (specifically checking for 'Enter' or Space), assign correct ARIA roles (e.g., `role="tablist"` and `role="tab"`), and verify that `focus-visible` styles are explicitly defined to support keyboard navigation.
