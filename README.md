# sprintLogic

Planificación inicial del producto antes de escribir código.

## Documentos base

- `docs/PROJECT_BLUEPRINT.md` — visión del producto, alcance MVP, objetivos y principios.
- `docs/ARCHITECTURE.md` — arquitectura objetivo, stack recomendado, módulos y modelo de datos.
- `docs/DEVELOPMENT_RULES.md` — reglas de desarrollo, calidad, testing y delivery.
- `docs/GIT_WORKFLOW.md` — flujo Git profesional, estrategia de ramas y reglas de commits atómicos.
- `docs/ROADMAP.md` — roadmap por fases, hitos y secuencia de ejecución.

## Enfoque

Este repositorio arranca con estrategia **documentation-first**:

1. Alinear visión, alcance y métricas.
2. Cerrar decisiones de arquitectura.
3. Definir reglas operativas de desarrollo.
4. Ejecutar MVP por fases con criterios de salida claros.

## Estándares de ingeniería

Este proyecto adopta desde su origen:

- **Clean Code**
- **Clean Architecture**
- **Git workflow profesional con ramas cortas y commits atómicos**
